accessibility The Cognos Schedule Panel

The Cognos Schedule panel allows you to create new Cognos Schedule Requests based on a series of event triggers (e.g. data availability) or modify those you have created earlier. The panel opens in a new browser window when you click the Schedule This Report icon for a schedule-enabled Cognos report or when you click on the Report Name link from the My Cognos Schedules page. Step-by-step instructions are provided and can be 'toggled' on and off by clicking on the 'Show instructions | Hide instructions' text. (Note that these schedules are separate from the time-based scheduling supported by the native Cognos product.)

The panel opens to the Edit prompts tab and also displays the Output format, e-Mail information, Schedule and Schedule history tabs.
Once the schedule has been set up, you may select a Priority to control the sequence in which your schedules are executed. The order of execution for a given trigger is:
  1. High priority requests
  2. Medium priority requests
  3. Low priority requests
  4. Off-peak requests

Note that Off-peak requests will be run when system resources allow them to complete without impacting the delivery of higher priority requests.

For reports that are configured to use the BI@IBM TBS Report Chaining function, additional Chained report controls options will be offered. These three settings allow you to control the run-time execution of the parent and child reports as follows:

  1. Run Parent and Child report - both schedule requests will be run (this is the default setting)
  2. Do not run/create Child report - only the Parent report schedule request will be run
  3. Do not run Parent report - only the Child report schedule request will be run

Before saving the updated schedule request by clicking on the Submit button, you may also ask for the report to be Run now. Simply click on the checkbox to add your request to the processing queue where it will execute irrespective of whether the trigger has been set that day or the specified schedule timing information. Note that if the schedule request was previously in Inactive state, the Run now request will reset it to Active.

Other functions include the ability to:

  • View any Existing schedules (My Cognos schedules page),
  • Create another schedule for the selected report.
  • Copy this schedule for yourself or for another BI@IBM user. Please note that all new copies for yourself will be set to Suspend state and will not execute until you update the selected prompts. Copies for another BI@IBM user will be set to Inactive state and will not execute until the recipient sets it to Active again.

accessibility Edit prompts tab

The Edit prompts tab enables you to define the prompts you need use in your scheduled report request.

  • Click on the Select prompts (new schedule) or Change prompts (existing schedule) button to retrieve the available prompts. (Note that the Cognos report design determines the prompts that may be used.)
  • Pick your desired prompts and click on Finish to return to the Cognos Schedule Panel. Your chosen prompts will be shown in the Your Selected Prompts box.
  • Add any Comments you require to help distinguish this request from any others you set up for this report.

accessibility e-Mail information tab

The e-Mail information tab is used to specify the recipient information for your Cognos schedule request.
Complete the following fields:

  • e-Mail address: This is a required field. You can type the e-Mail intranet address for up to 12 recipients separated by spaces. You may use a combination of individual IDs and any report distribution lists you have already set up using the Report distribution lists link in the Resource center portlet.
     
  • e-Mail subject: This is a required field. Type the text to display in the subject line of the e-Mail. If you do not enter text, the report name displays by default. Note: It is recommended that you use this field to make it easy to identify this specific request in your mail.
     
  • e-Mail comments: This is an optional field. Type comments about the report that you want to include in the body of the e-Mail message.
     
  • Backup: This is an optional field and may be used to enter the intranet ID of another BI@IBM users who be given edit access to this schedule request. Note that this status is indicated in the backup owner's My Cognos schedules page by a 'B' in the Owner column.
     
  • Mail options: These selections allow you to determine if the report output is to be e-Mailed to its target recipients or just left on the system server for use in Autodeck processing. If the e-Mail option is selected, additional Mail type options determine whether the output file itself is to be included in the e-Mail message and/or whether a link (url) to the report is preferred.
    The Show SMS SOM Support Team setting modifies the e-Mail subject line and content, and is intended to be used in those circumstances when the SMS team has scheduled the report on behalf of their stakeholder. If the Yes option is selected, the Requested by section of the e-Mail subject line is replaced by the text "Scheduled on your behalf by the SMS SOM Support Team" and a link is added to the Support Team application in place of the default link enabling the owner to edit the related Cognos Schedule Request.
    The final mail option determines whether or not any selected output file should be sent in compressed format (i.e. zip file).
     
  • Notify errors?: Regardless of the selected Send mail option, you may choose to receive e-Mail notifications of any associated schedule processing errors.

accessibility Schedule tab

The Schedule tab is used to specify the event trigger and requested timing for your Cognos schedule request. Note that unless the Run now check-box is selected when you Submit your request, the report schedule will not run until the data load trigger is set and the selected day arrives.
Complete all the following required fields:

  • Schedule frequency: Different options are presented for different trigger types - select the option most appropriate to your needs:

    • Weekly Triggers
      • Weekly - will run as soon as the requested weekly data is available.
      • Selected day of the week - will run on the selected day if the data is available; if not the schedule will wait for that week's data trigger.
      • Monthly - will run on the selected day(s) of the month using the then available weekly data.
      • Business event - will run as soon as the designated event occurs using the then available weekly data. This is useful for the reports such as the QSAI reports that are only required at specific times during the business cycle.
    • Daily Triggers
      • Daily - will run using close of business data for the selected day(s).
      • Monthly - will run using close of business data for the selected day(s) of the month.
    • NRT Triggers
      • Not applicable - report will only be run when the Run now option is selected.

  • Data load timing: Select the data load update trigger for this schedule request. (Note that this option is not applicable for NRT report schedule requests.)
     
  • Expiration date: Specify when the report schedule expires - the date will default to the maximum allowed for the selected schedule frequency. You will be prompted to extend the Expiration date before the schedule is set to 'Inactive' in case you still need it to run. (Note that this option is not applicable to NRT report schedule requests.)

accessibility Schedule history tab

The Schedule history tab displays a summary of the execution status of this schedule but will only appear after your schedule request has run for the first time.
The information provided includes:
  • Start time and Finish time (date and time stamp - GMT)
  • Status code and explanatory message. In the event of failures please click on the Running ID link for additional information. If the report ran successfully (code 100 or 101) the link will attempt to download that report output.
    • 100 - Report ran successfully.
    • 101 - Report ran and sent out mail successfully.
    • 111 - Report ran successfully but the report output is no longer available for download.
    • 200 - Report failed to run successfully.
    • 300 - Report is running.
    • 400 - Timeout
  • Flags indicating whether the e-Mail was sent and whether the report output was published for broader usage
