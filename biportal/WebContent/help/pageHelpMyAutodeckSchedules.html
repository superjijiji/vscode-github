The My Autodeck schedules page enables you to manage the Autodecks that you have already created and provides information regarding the last attempt to create each Autodeck.  

Open the Existing Autodecks page by clicking on My Autodeck schedules in the BI@IBM controls portlet. A series of links are provided in the left hand navigation panel to allow you to invoke other BI@IBM functions. These links are:

  • Create an Autodeck - opens a new page where a new Autodeck definition can be created.
  • Manage xls inputs - provides mechanisms to allow you to upload and maintain your own Excel files (.xls or .xlsx) that can then be included in an Autodeck request alongside the TBS output files you have access to.
  • My Cognos schedules - enables you to manage the Cognos trigger-based report schedules that you have created.
  • My distribution lists - invoke the BI@IBM function to create and maintain a distribution lists containing the intranet IDs of BI@IBM users and which can be used as recipients of Autodeck and Cognos schedule request outputs.

The Existing Autodecks Information Display

The My Autodeck schedules page opens to the Existing Autodecks view which can be used to manage your existing presentation decks. The information that is displayed can be customised by using the Show/Hide columns button. Note that the Checkbox, Deck ID, Execution Status and Last Run Date columns cannot be hidden.

  • Deck ID - provides a link to the Autodeck schedule request. To change the settings for a schedule, click the link in the Deck ID field. This action opens the saved Autodeck request where you can update the requested deck creation information, required content and associated delivery information. When your changes are complete, click on Submit to save your changes and return.
  • e-Mail Subject - shows the text that will appear as the Subject line when the Autodeck output is mailed to its target recipients.
  • e-Mail Address - shows the list of recipients for the Autodeck output.
  • Comments - text added as e-Mail comments on the e-Mail information page in the Autodeck request up. These are included in the body of the e-Mail sent to the recipients of this Autodeck.
  • Frequency - shows the days of the week that the Autodeck should be generated. Note that unless a Provisional deck has been requested, the process will only run on the selected days if all required TBS inputs are Ready.
  • Expiry - the last date that this schedule request will be run. Note that you will be prompted to extend the Expiration date before the schedule is set to 'Inactive' in case you still need it to run.
  • Status - shows whether the request is Active (i.e. will be run) or Inactive.
  • Type - indicates whether the Autodeck output will be a PowerPoint or an Excel file.
  • Owner - indicates if you are the Owner of the schedule request or are the designated Backup.
  • Execution Status - provides information regarding the most recent Autodeck generation and acts as a link to download the most recent output file. A more complete Schedule history is available in the Autodeck definition itself. There are four status values that can be displayed, each of which will be shown with a suffix of (F) or (P) to indicate whether the last output was Final or Provisional. Additional information is provided as 'bubble text' if you hover your mouse pointer over the Execution Status value:
    • Running - the Autodeck generation is currently in progress.
    • Success - the Autodeck was successfully produced.
    • Warning - the Autodeck output was created but some of the requested content was unavailable.
    • Failed - system or configuration errors prevented the Autodeck generation from completing successfully.
    Note: if the Execution status shows that your Autodeck has been generated (Success or Warning) but you have not received your output within the hour, please contact the BI@IBM Help Desk for further assistance.  
  • Last Run Date (GMT) - shows the date in GMT when the schedule was last run.

Notes: Clicking on any of the column headings will sort the displayed information alphabetically. Clicking on that column header again will switch the display order. An icon will appear next to the column heading to indicate whether this is in ascending or descending order.

The Search box allows you to locate the subset of displayed Autodeck requests that match your search criterion. The display will update dynamically as you enter a text string (2 or more characters) and will display only those requests that contain the designated text.

Working with the displayed Autodeck Schedule Requests

Select the check boxes for the report schedules for which you want to perform the following actions and then click on the relevant button.

  • Show / Hide columns - click to open the dialog to customise the information that is displayed and that is available to the on-page search function. Select the required items by clicking on the Yes or No buttons. Note that the Checkbox, Deck ID, Execution Status and Last Run Date columns cannot be hidden.
     
  • Show / Hide links - click to toggle between showing the left hand navigation links and using the full width of the page for the selected Autodeck information.
     
  • Show selected / all - click to toggle between showing the subset of schedule requests you have selected and the full set available in the current view.
     
  • Submit now - click to request immediate creation of the selected deck with the most current content. 
    Important notes:
    • all of the Cognos schedule requests that you use must have been run successfully at least once and all of the uploaded files must be available on the system for the Autodeck to be created;
    • the Autodeck must be in an Active state or still be in the Grace period following automated expiry before it can be created using this process.
    • the e-Mail subject will indicate that the attached Autodeck is "Provisional" - even if you have not indicated that a Provisional deck is required on the Schedule tab.
    • the Reset Output Status function on the Edit Requests tab of your Existing Autodeck definitions allows you to resync the TBS inputs by toggling between the Ready and Not Ready states for the selected TBS input(s) if necessary.

    •  
  • Activate/inactivate - click to switch between active and inactive. Inactive schedules do not run but remain in the Schedules list and can be reactivated to run again later.
     
  • Extend dates - click to extend the expiration date to the maximum allowed. The Expiration Date column displays the new date. You may click on the date to download the most recent Autodeck output.
     
  • Bulk update - click to invoke the following additional functions:
     
    • Activate/inactivate - click to switch between active and inactive. Inactive schedules do not run but remain in the Schedules list and can be reactivated to run again later.
    • Copy selected Autodeck schedules - click on the menu option to create a new copy of the selected Autodeck. A new window will appear giving you the option to copy the definition yourself or to another specified user (along with all the required Autodeck inputs). Note that the new deck will have the text '- Copy' appended to the e-Mail subject and will initially be set to Inactive.
    • Submit now - click to request immediate creation of the selected deck with the most current content.
    • e-Mail information - Change back-up - lets you change the designated Back-up owner on all selected schedule requests. If you want to remove the Back-up just leave the owner field empty.
    • e-Mail information - Change e-Mail addresses - opens a dialog to let you change the e-Mail recipients for the selected Autodeck outputs.
    • e-Mail information - Change e-Mail comments - lets you change the comments included in the body of the selected Autodeck e-Mails.
    • Output format - Change deck file name - update the chosen file name for the selected Autodecks as specified on the Output format tab
    • Output format - Append date options - modify selections regarding the addition of date information to the Autodeck output file name.
    • Output format - Choose link or attachment - update the file distribution option for the selected Autodecks as specified on the Output format tab
    • Schedule - Change final deck synchronisation - modify the timing options controlling the automatic generation of your request Autodecks as specified on the Schedule tab.
    • Schedule - Change message settings - update the chosen Provisonal Deck / Status message options.
    • Schedule - Extend date - click on the menu otion to extend the expiration date to the maximum allowed.
       
  • Delete - click to delete the Autodeck schedule requests.
