The Cognos Schedule Status page helps you understand the current state of all requested Cognos inputs for the selected Active Autodeck definition.  
 
In addition to showing the date and time (GMT) that the last Final deck was produced, this page provides a table of the associated Autodeck inputs with the following information:  
  • Report Name - provides a link to the most recent Cognos Schedule Request (CSR) output file.
  • e-Mail Subject - shows the text that will appear as the Subject line when the report output is mailed to its target recipients.
  • Frequency - shows the days of the week that the schedule can be triggered. Note that the report will only run on these days regardless of when the trigger is actually received.
  • Ready for Final Deck - indicates whether the Autodeck inputs are currently ready for a Final deck to be created:
    • Ready - the CSR has been updated since the last Final Autodeck was produced
    • Failed - the last attempt to run this CSR failed.
    • Unavailable - a new CSR input that has never been run.
    • Not Ready - the CSR has not been updated since the last Final Autodeck was produced.
    • Inactive - the CSR has been set to Inactive state and will not be triggered.
    • Suspend - BI@IBM administrators have stopped this report from running due to system issues.
    • Deleted - the CSR can no longer be run as the report is no longer available
    Note that immediately after the Autodeck is run, all inputs will normally show Not Ready unless one or more have been run again in the intervening period.
  • Execution Status - information regarding the status of most recent attempt to run this schedule request:
    • 100 - Report ran successfully.
    • 101 - Report ran and sent out mail successfully.
    • 200 - Report failed to run successfully.
    • 300 - Report is running.
    • 400 - Timeout
  • Comments - text added as comments on the edit prompts page when you set the schedule request up. This information helps you distinguish between multiple schedules for the same report.
  • Last Update Time - shows the date and time (GMT) when the schedule was last run.
  • Request ID - the unique BI@IBM identifier for the specified Cognos Schedule Request.
