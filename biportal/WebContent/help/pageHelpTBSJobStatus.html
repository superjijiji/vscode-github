The TBS Execution Status page helps you understand the current state of the BI@IBM Trigger-Based Schedule workload.  
 
Use the menu option to select the relevant Cognos domain to be displayed. The on-page search function may also be used to help identify relevant TBS job streams. Just type the desired text string in the Search box to restrict the display to all entries containing that string. The following information is provided:  
  • Run Date - the date (GMT based) that the trigger was set.
  • Trigger - the name of the trigger as shown in the Cognos Schedule Request and the BI@IBM Data load status portlet.
  • TBS Run - a number indicating whether this is the 1st, 2nd or 3rd (final) attempt to run these schedules
  • Start Time (GMT) - the time this particular TBS run was started.
  • End Time (GMT) - the time the specified TBS run was completed.
  • Success - the number of schedules that have been successfully completed.
  • Avg Run Time - the average time taken to complete the schedules shown in minutes and seconds.
  • Active - the number of schedule requests currently being executed.
  • To Go - the number of outstanding schedule requests for this run.
  • Fail - the number of schedule requests that have failed to run successfully
  • Off-peak Included - Y/N flag indicating whether schedules with an off-peak priority are included in this TBS run.
  • Status - shows the execution status for the specified TBS run:
    • Done - TBS run has completed and information is current.
    • Terminated - TBS run was cancelled.
    • In-progress - TBS run is in progress. To refresh the status of an active TBS job click on the In-progress link
    • On-hold - TBS run has been temporarily suspended.
    • Out of Date - TBS run was completed or terminated but the information is no longer current.
    Note: each row in the table is color-coded to indicate whether the designated TBS Run is:
    • In-progress or On-hold - Yellow
    • Completed today - Green
    • Completed yesterday/last week - Red
