The My Cognos schedules page enables you to manage the Cognos trigger-based report schedules that you have created.  

Open the page by clicking on My Cognos schedules in the BI@IBM controls portlet. A series of links are provided in the left hand navigation panel to allow you to invoke other BI@IBM functions. These links enable:

  • Easy access to the BI@IBM My Autodecks and My distribution lists pages.
  • Reporting of the overall BI@IBM TBS execution status across all data load triggers and Cognos domains.
  • Direct publication of Cognos Schedule outputs to the BI@IBM Report Library. Note that this function is enabled for Authorised users only.
  • Working with the associated Cognos Schedule Requests By Autodeck. A new panel will allow you to select the Autodeck whose schedules you then wish to work with.
     

The Information Display

The My Cognos schedules page opens to the All my schedules view. If you have created folders to help manage your Cognos Schedule Requests (see Manage folders below), the folder names will be shown immediately below this default link in the left hand navigator. Clicking on a folder name will limit the information displayed to that subset of your request portfolio.

The information that is displayed can be customised by using the Show/Hide columns button. Note that the Checkbox, Report Name, Run Status and Run Date columns cannot be hidden.

  • Report Name - provides a link to the Cognos schedule request. To change the settings for a schedule, click the link in the Report Name field. This action opens the saved Cognos schedule request page for that report where you can modify the schedule settings. When your changes are complete, click on Submit to save your changes and return.
  • e-Mail Subject - shows the text that will appear as the Subject line when the report output is mailed to its target recipients.
  • e-Mail Address - shows the list of recipients for the report output.
  • Comments - text added as comments on the edit prompts page when you set the schedule request up. This is intended to help you distinguish between multiple schedules for the same report.
    Note: if you see the text (System Generated Schedule) appended to the comment text, this implies that the schedule request has been automatically created for you using the prompts and other schedule execution parameters that you defined on an associated parent report. This TBS Chaining feature is normally used when a summary report has related detail section that has been designed as a separate report for performance reasons. The System Generated Schedule will be recreated and rerun using the latest execution parameters every time that the parent report is triggered unless you specifically disable it by setting it to Inactive or update the Chained report controls options within the parent report schedule request.
  • Frequency - shows the days of the week that the schedule can be triggered. Note that the report will only run on these days regardless of when the trigger is actually received.
  • Data Load - the chosen data load trigger and its current status. OK (green text) means the data source is online and available for schedules to be run; NO (red text) means the data source is currently unavailable.
  • Expiration Date - the last date this schedule request can be run. Note that you will be prompted to extend the Expiration date before the schedule is set to 'Inactive' in case you still need it to run.
  • Status - shows whether the request is Active, Inactive, Suspended, Disabled or Expired. An additional (C) designation indicates that the request is a system generated Chained report. The Active/Inactive status is under user control whereas Expired, Suspended and Disabled are determined by system Administrators. Schedule requests may be temporarily Suspended from TBS execution because of report design changes that require prompts to be reselected. Once the prompts are updated, the CSR may be set to Active again. They may be Disabled because the reports are no longer available on the system. In this latter case you should select the designated CSRs and click the Delete button to remove the associated Cognos Schedule Requests.
  • Send Mail - shows whether the request output is to be mailed to its target recipients (Y) or not sent at all (N).
  • Priority - shows the relative execution priority of the request within your portfolio of Cognos schedules.
  • Owner - indicates if you are the Owner of the schedule request or are the designated Backup.
  • Run Status - shows the completion status code from the last attempt to run the schedule. Note that successes (codes 100 and 101) are shown in green whereas error codes are displayed in red text.
    • 100 - Report ran successfully.
    • 101 - Report ran and sent out mail successfully.
    • 111 - Report ran successfully but the report output is no longer available for download.
    • 200 - Report failed to run successfully.
    • 300 - Report is running.
    • 400 - Timeout
  • Run Date - shows the date when the schedule was last run.

Notes: Clicking on any of the column headings will sort the displayed information alphabetically. Clicking on that column header again will switch the display order. An icon will appear next to the column heading to indicate whether this is in ascending or descending order.

The Search box allows you to locate the subset of displayed requests that match your search criterion. The display will update dynamically as you enter a text string (2 or more characters) and will display only those requests that contain the designated text.

Working with the displayed Cognos Schedule Requests

Select the check boxes for the report schedules for which you want to perform the following actions and then click on the relevant button.

  • Show / Hide columns - click to open the dialog to customise the information that is displayed and that is available to the on-page search function. Select the required items by clicking on the Yes or No buttons. Note that the Checkbox, Report Name, Run Status and Run Date columns cannot be hidden.
     
  • Show / Hide links - click to toggle between showing the left hand navigation links and using the full width of the page for the selected Cognos Schedule Request information.
     
  • Show selected / all - click to toggle between showing the subset of schedule requests you have selected and the full set available in the current view.
     
  • Run now - click to add the selected report schedule request to the job queue. Please note that the CSR will be set to Active following this request regardless of its original state.
     
  • Bulk update - click to invoke the following additional functions:
     
    • Activate/inactivate - click to switch between active and inactive. Inactive schedules do not run but remain in the Schedules list and can be reactivated to run again later.
       
    • Change comments - lets you change the schedule request comments. (Note that these are not the e-Mail comments.)
       
    • Change e-Mail addresses - opens a dialog to let you change the e-Mail recipients for the selected TBS request outputs.
       
    • Change e-Mail comments - lets you change the comments included in the selected TBS e-Mails.
       
    • Change Error Notifications - lets you choose whether to be notified of errors in the selected TBS requests.
       
    • Change Data Load Timing - lets you the change the data load timing to be used for the selected TBS requests. Note that this function only works for schedule requests based on the same report.
       
    • Change Frequencies - opens a dialog to let you change the required schedule frequency on all selected requests. Note that this function will only work if the selected requests are all associated with the same data load trigger.
       
    • Change Mail options - opens a dialog to let you modify the various mail options for the selected requests:
      • 'Send Mail' status - 'Y' (Send output) or 'N' (Don't send).
      • Select the Mail type (Output file, url for downloading or both).
      • 'Compressed file' status - 'Y' (zipped) or 'N' (uncompressed).
      • 'Show SMS SOM Support Team' info - 'Y' (show) and 'N' (Don't show).
         
    • Change Output Formats - lets you modify the output formats for the selected TBS requests. Note that this function only works for schedule requests based on the same report.
       
    • Copy selected schedules - dialog will open prompting you to enter the target recipient's intranet ID. By default the copy will be for your own use. If you enter another ID, both you and the recipient will recieve confirmation e-Mails when the operation successfully completes. Note that new copies for yourself will initially be set to SUSPENDED state. You should make all required modifications, re-select the prompts and then submit the schedule. After the successful update, your new copy will be set to ACTIVE state. Copies sent to another user will be set to Inactive state and will not run until they are set to Active by using the Activate/inactivate button.
       
    • Extend dates - click to extend the expiration date to the maximum allowed. The Expiration Date column displays the new date.
       
    • Manage back-ups - lets you change the designated Back-up owner on all selected schedule requests. If you want to remove the Back-up just leave the owner field empty.
       
    • Manage priorities - opens a dialog to let you change the comments included in the selected TBS e-Mails.
      • High Priority - set the selected schedule request(s) to High priority.
      • Medium Priority - set the selected schedule request(s) to Medium priority.
      • Low Priority - set the selected schedule request(s) to Low priority.
      • Off-Peak - set the selected schedule request(s) to Off-Peak priority.
         
    • Take ownership - lets the Back-up owner take ownership of the selected schedule request(s). After this update the previous Owner will have no access to the request but will still be copied on the schedule output.
       
    • Toggle back-up - this function swaps the Owner and Back-up IDs associated with the CSR. After this update both the IDs continue to have access to the schedule request.
       
  • Download outputs - click on the Run date link to download the most recent output file for that schedule request or select the CSRs and then click on the Download outputs button. If you want all available output files, you may click on the checkbox alongside the table column headings to select all rows. A checkbox is provided to allow you to request that the selected outputs are downloaded as a compressed (i.e. zipped) file if required. Note that the unique Request ID will be appended to the e-Mail subject to help you distinguish multiple outputs with what may otherwise have the same name. To minimise any possible confusion, please ensure that you use unique e-Mail subjects across all of your Cognos Schedule Requests. Note also that output files are only retained for two weeks - schedules that were run before that time will not be available for download until they are rerun.
     
  • Manage folders button to create your desired folder structure to help organise your Cognos Schedule Requests
    • Enter the desired folder name in the text box and then click on the Create folder button to create the folder.
    • Select the CSRs you require and drag them to the new folder (a green check mark will indicate when they can be dropped in place) or use the Move selected button.
    • To rename a folder, enter the new folder name in the text box and then highlight the folder to be renamed before clicking on the Rename folder button.
    • To delete a folder, highlight the folder to be deleted before clicking on the Delete folder button.
    • To re-sequence folders or change the display order of CSRs within a folder highlight the desired content and drag and drop it in the new location.

      IMPORTANT NOTE - You must click on the Save changes button before clicking on the Back link to return the My Cognos schedules page.
     
  • Delete - click to delete the report schedule.
